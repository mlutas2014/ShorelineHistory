Introduction

Research on how sea level change affects coastline position requires objective measurement of coastlines over time. This can be done by comparing coastline positions as digitised from aerial photos of the coast taken as snapshots at intervals over time. This tool measures change in shoreline position over time along a set of equally spaced coastal transects.

ShorelineHistory has a graphical user interface for entering input parameters and to run the geoprocessing steps. At the core of the tool is a set of Python scripts for geoprocessing in ArcGIS. The tool produces tables of data showing the coastline positions at transects and changes in position along transects. Users can then export the data for further analysis in Excel or other software.

Installation Instructions

The ShorelineHistory application and associated files can be found on Github.com at ShorelineHistory

The application and associated files is available as a zipped archive. Just extract the archive into an appropriate place on your computer.

In the same folder as the ‘ShorelineHistory.exe’ file the ‘bin’ folder is where the Python scripts are stored. The ‘bin’ folder needs to be kept in the same folder as the ‘ShorelineHistory.exe’ file. Also, for correct operation, the scripts in the ‘bin’ folder should not be modified or renamed.

An example dataset is also included.

System Requirements

ShorelineHistory is Windows executable (exe) application that uses ArcPy for geoprocessing. It is not an ArcGIS extension. ShorelineHistory is a .NET application using .NET framework version 3.5. It requires ArcGIS 10 or higher and has been tested on ArcGIS versions up to 10.5.

Data Formats

The shapefile format is used for all spatial data. Spatial data is required to be in a projected coordinate system with measurements in metres. Users can export output data to other software, such as Excel, for analysis. Some example input datasets are included with this application.

 

Description of spatial datasets used in or created by ShorelineHistory

Digitised coastline shapefiles

These are the coastlines for which positions will be measured at transects. Input coastlines can be prepared in a number of ways such as by digitisation from ortho-rectified aerial photos or as a height contours from a 3D datasets. When digitising coastlines from aerial photos it is important to be consistent in the choice of the surface features associated with where the coastline is drawn. For example the vegetation line may be chosen as the coastline above which vegetation is present.

All input digitised coastlines need to be separate shapefiles and need to be placed within the same folder (input shorelines folder). The coastlines can be in multiple sub-folders within that folder. All need to be projected in the same coordinate system which has measurements in metres. The name of the folder will be used to identify the coastline in the output data. Digitised coastline names need to begin with the coastline name which is at least three letters long and need to end in year, month and day as an eight digit number in their filenames. For example Bondi20150429.shp is a Bondi beach coastline as digitised for 29th April 2015. All shapefiles in the folder should follow the same format. The position of each coastline needs to be such that they intersect with each transect.

An example digitised coastline is used by tools on the first tab for establishing processing extent. This can also be the input coastline used for producing the coast points file.

Coast polygon shapefile

A polygon version of the land adjacent to the coastline. This polygon needs to extend inward beyond at least the landward extent of each transect. The seaward end of each transect also needs to be outside of the polygon. The purpose of the coast polygon shapefile is to determine which end of each transect is the inshore end.

Reference coastline shapefile

This is a line version of the reference coastline and is used as reference coast position for generating transects. Transects will be centred along this line. This can also be the input coastline used for producing the coast points shapefile. Note: Because of the way transects are generated, transects are not created for the points at either end of the reference points shapefile. Therefore the reference coastline needs to be longer at both ends, than the coastline to be assessed, by at least the spacing between two transects.

Coast points shapefile

A shapefile containing numbered points at equal spacing (eg. 100 m) along the coastline. This shapefile will have an attribute that uniquely identifies each point (ID field name). The data type for the coast points ID Field needs to be integer. The coast points shapefile is used to determine the centre position of each transect and to provide the unique ID code for each transect. If no coast points file is available this can be created from a coastline using a script run from the 'Coast points' tab.

Transects shapefile

The transects are the reference lines perpendicular to the coast used to measure coastline change. This shapefile is first generated by the 'Make transects and vertex points shapefiles' tool on tab 1. Each transect will share the same ID number attribute from the coastline points shapefile and transect vertex point shapefile.  After creation, this file needs to be checked and edited before proceeding to subsequent processing steps.

Transect Vertex points shapefile

This shapefile is generated by the 'Make transects and vertex points shapefiles' tool on tab 1. The inside vertex point is at the most landward end of each transect. Each point will have the same ID number as the associated transect and the associated point from the coast points file. The transect vertex point shapefile is used by ShorelineHistory to define the landward end of each transect and the zero point for measurement of the position of each intersecting coastline.

 

Notes on creating a reference coastline and reference points file.

The reference coastline is used to define the analysis extent along the coastline and should be approximately centrally positioned along the coastal contour relative to the digitised coastlines. A new reference points shapefile can be created from a reference coastline shapefile. The reference points shapefile is in turn used by ShorelineHistory to create the transects shapefile. Because of the way transects are generated, transects are not created for the points at either end of the reference points shapefile. Therefore the reference coastline needs to be longer at both ends, than the coastline to be assessed, by at least the spacing between two transects.

 Also if the length of the reference coastline is not an exact multiple of the spacing between transects the last one will be spaced at a smaller distance. For example if 1020 metres of coastline, with points at 100 metre spacing, the last one will be just 20 metres away from the end. This is not usually a problem since the end two points are not used (as noted above) to make transects anyway but in some cases it might put the end transect at the wrong spacing. So in short, make the reference coastline a little longer than needed.

Work flow

To run the application, double-click (or right-click and select ‘open’) the ShorelineHistory.exe file or a shortcut.

The ShorelineHistory application opens as a tabbed form. Individual tabs can be displayed by clicking on the tab headings. Help information will be displayed if the ‘Show help’ check box has been ticked.

Clicking on text boxes will reveal help information on what is needed to be input. Moving the cursor over tool buttons will also reveal help information on the button function. Note, when clicking on filenames within text boxes the text colour will change to blue. After then clicking elsewhere on the tab the text will change to black if the file exists or to red if the file does not exist.

The first four tabs are in workflow order.

1.       The first tab is concerned with preparing inputs.

2.       The second tab takes the inputs prepared using the first tab and generates a set of shapefiles.

3.       The third tab extracts data from the output shapefiles generated with the previous tab.

4.       The fourth tab is used to sort data generated with the previous tab as shorelines in date order.

 

Preparing inputs

The first tab ‘Prepare inputs’ is used to generate two shapefiles which are needed as inputs for calculating shoreline change. These shapefiles are the transects shapefile and the point file to identify which end of each transect is landward (transect vertex point shapefile). The tab contains text entry controls for entering input parameters and the intended names and file locations of the output files. There are three buttons to run the three geoprocessing tools in sequential order. The following flowchart diagram shows inputs and outputs (blue) and the geoprocessing steps (orange) for these tools. Arrows show data flow. Before running tools, first make sure that none of the input shapefiles or files with the same name as outputs are open in ARCmap or other application. This is to avoid schema lock conflicts which will halt processing.

 

1 Make transects and vertex points shapefiles

This tool generates an initial set of transect and inside vertex point shapefiles. After creation, the transects file needs to be checked and edited as follows.

1.1 Editing following initial creation of transects and vertex points shapefiles

Close ShorelineHistory  before editing shapefiles in ARCmap to avoid schema lock conflicts. The output transects shapefile needs to be edited after creation to:

·         remove transects that are outside the intended target area

·         remove or edit any crossed transects

·         extend any transects that do not intersect with the reference coastline

At this stage there is no need to edit the transects attribute table or vertex points shapefile as they will be regenerated using tools 2 and 3 on tab 1.

The following image shows the initial transects (green) and inside vertex points (red) generated from a set of test data. The points are labelled with the reference number. Also shown is the reference coastline (violet) the land polygon and a boundary line showing the processing extent.

In the example the intended transects were to have reference numbers sequentially from 166621 to 16628. Additional transects outside the target region (circled below in blue) needed to be deleted.

Sometimes transects may cross and join, producing a Y shaped double transect or other configuration. If they are outside the target area (as in the above example) they can be deleted. If they are inside the target area the individual transects need to be edited as individual non-intersecting lines. Each edited transect needs to cross its associated reference point (tolerance of within 15 metres) and can be of arbitrary length sufficient to cross all of the digitised coastlines. If transects are crossing each other an alternative may be to rerun the tool with a shorter transect length.

Each digitised coastline needs to intersect with each transect.  If any of the transects do not cross a digitised coastline then at this stage the relevant transects can be edited, extending them to cross all coastlines. Each edited transect also needs to cross its associated reference point (tolerance of within 15 metres).

After editing, close ARCmap to avoid schema lock conflicts.

 

2 Update transects attributes

After editing the transects the transect attributes need to be updated. The ‘Update transects attributes’ tool can be used to ensure that the attribute table is correctly regenerated. In this step the transect reference number field is refreshed, taking the reference number from the associated reference point.

3 Recalculate inside vertex point shapefile

After editing the transects and updating transect attributes the inside vertex shapefile also needs to be updated. The ‘Recalculate inside vertex point shapefile’ tool can be used to ensure that this shapefile is correctly regenerated.

 

Calculating Shoreline Change

The tool for calculating shoreline change is on the ‘Calculating Shoreline Change’ tab. All the inputs for running this tool will need to have been previously generated using tools located on the ‘Prepare Inputs’ tab.

Before running this tool, make sure that none of the input shapefiles or files with the same name as outputs are open in ARCmap or other application. To run this tool first enter the parameters then click the ‘Process shorelines’ button.

The tool will generate two shapefiles for each of the intersecting coastlines. These are an intersection points shapefile and a shortened transects shapefile. The shortened transects files are named in the following format : first three characters of the coastline name + “_” + coastline date as YYYYMMDD, (for example ‘Kin_20050628’). The same format is used for the points shapefile except that ‘_Points’ is appended to the name, (eg. ‘Kin_20050628_Points’). These will be in two new folders (‘Points’ and ‘ShortenedTransects’) in the output folder.

Output shapefiles can be viewed in ArcMAP. The listed attribute data can also be viewed and exported as text using the next two tabs.

 

Listing output data

After calculating shoreline change the output data can be viewed in the ‘Output data’ tab.

Initially only the ‘List Data’ button will be enabled. Click this to extract data from the output shapefiles.

 

This data can now be saved to a .csv text file using the ‘Save to file’ button.

Sorting data by date

After extracting data using the ‘Output data’ tab you may wish to view the distances sorted by date and transect number. This can be done on the ‘Sorted by date’ tab. Click on the ‘List data by date’ button.

This re-sorts the data from the previous tab so that for each transect and shoreline name the associated lengths are listed by date. Dates will appear in the table in YYYYMMDD format but when saved they will be saved (‘Save to file’ button) in DD/MM/YYYY format. Note that the data in the previous tab needs be already calculated.

These lengths are the distance from the transect inside vertex. When saved the output .csv file will also include number of intersecting transects (n), median, minimum, maximum and mid-range values for each transect. Data can then be viewed in Excel or other application as shown below.

 

Generating a coast points shapefile

If no coast point file exists or if a different point spacing is required then this tool can be used to create a coast reference point file with points at the required spacing. The ID field is used to identify individual points. When making transects (prepare inputs tab) it is recommended that the coast buffer size be about half the point spacing otherwise there is a greater risk that transects could intersect.

 

 

Script log tab

The final tab of ShorelineHistory is the ‘Script log’ tab. This has two windows: the ‘Script log’ and ‘Script error log’. This tab will show script outputs after running any of the geoprocessing tools on the first two tabs or the ‘make coast points shapefile’ tool on the ‘Coast points tab’. Script log shows tool processing details where the tool has run successfully. If a geoprocessing error has occurred this will usually show up as a message on 'Script error log'.

Setting the ARCGIS Python installation path

There may be multiple Python installations on your computer and sometimes the wrong Python installation may be set as a default. At application start-up ShorelineHistory attempts to set the correct Python path which should be something like ‘C:\Python27\ArcGIS10.5\python.exe’. If the correct path cannot be set the path defaults to ‘python’ which can sometimes work if there is only one Python installation on the computer or the last one installed was the ArcGIS installation.

The correct path can be set by clicking the ‘Browse’ button next to the ‘Set alternative python application’ text box and browsing first to the Python folder (eg. Python27) then within that to the ArcGIS version folder (eg. ArcGIS10.5) and then within that clicking on python.exe. Alternatively if you know the correct path you can just paste it into the text box.
